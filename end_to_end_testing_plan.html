1. Authentication & Authorization (19 Tests)

1.1 User Registration

Frontend: /register | Backend: POST /api/auth/register

TC-AUTH-001: Register with valid email, name, and password
Steps: Navigate to register page → Fill form → Submit
Expected: User created, redirected to dashboard, JWT token stored
Backend: User document created in MongoDB, password hashed with bcrypt
TC-AUTH-002: Register with existing email → Error "User already exists"
TC-AUTH-003: Register with weak password (< 6 chars) → Validation error
TC-AUTH-004: Register with invalid email format → Email validation error

1.2 User Login

Frontend: /login | Backend: POST /api/auth/login

TC-AUTH-005: Login with valid credentials → Successful login, JWT stored
TC-AUTH-006: Login with incorrect password → "Invalid credentials" error
TC-AUTH-007: Login with non-existent email → "Invalid credentials" error
TC-AUTH-008: Case-insensitive email login → Success regardless of case

1.3 OTP-Based Login

Backend: POST /api/auth/send-otp, POST /api/auth/verify-otp

TC-AUTH-009: Send OTP to valid phone → OTP sent, stored with expiry
TC-AUTH-010: Verify correct OTP → Login successful, JWT issued
TC-AUTH-011: Verify incorrect OTP → "Invalid OTP" error

Testing Execution Guidelines

Backend Testing

  1. API Testing Tools: Use Postman or Thunder Client
  2. Database Verification: Check MongoDB documents after operations
  3. Authentication: Include JWT token in Authorization header
  4. Error Handling: Test both success and error scenarios

Frontend Testing

  1. Browser Testing: Test on Chrome, Firefox, Safari
  2. Responsive Design: Test on mobile, tablet, desktop
  3. Network Conditions: Test with slow 3G, offline
  4. Console Errors: Check browser console for errors

Integration Testing

  1. End-to-End Flows: Complete user journeys
  2. Role-Based Access: Test as user, admin, delivery agent, guest
  3. Data Consistency: Verify frontend-backend sync

Performance Testing

  1. Page Load Times: Measure initial load and navigation
  2. API Response Times: Check endpoint response times
  3. Database Queries: Monitor query performance

Security Testing

  1. Authentication: Test token expiration, invalid tokens
  2. Authorization: Test role-based access control
  3. Input Validation: Test SQL injection, XSS attempts
  4. Password Security: Verify bcrypt hashing
